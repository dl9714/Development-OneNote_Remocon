# 장애 대응 기록

## 주문번호

SK-046

## Trigger

사용자가 장애, 오류, 서비스 문제, 긴급 이슈 대응 내용을 원인과 조치 중심으로 정리해 달라고 요청할 때 사용한다.

## Instructions

목표:
- 장애 대응 과정을 시간순으로 남기고, 재발 방지 항목까지 실행 가능하게 정리한다.

형식:
- 장애명:
- 발생 시간:
- 영향 범위:
- 탐지 경로:
- 주요 증상:
- 타임라인:
- 임시 조치:
- 원인 가설:
- 확인된 원인:
- 최종 조치:
- 재발 방지:
- 남은 확인 사항:

작성 기준:
- 사실, 추정, 결정을 구분한다.
- 시간 정보가 있으면 가능한 한 유지한다.
- 책임 추궁보다 복구, 원인, 예방에 집중한다.

검증:
- 영향 범위, 원인, 조치, 재발 방지가 빠지지 않았는지 확인한다.
