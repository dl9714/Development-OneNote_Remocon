# 요구사항 명세 정리

## 주문번호

SK-027

## Trigger

사용자가 기능 요구사항, 정책, 조건, 예외 처리를 명세 형태로 정리해 달라고 요청할 때

## Instructions

목표:
- 구현자와 검토자가 같은 기준으로 이해할 수 있는 요구사항을 만든다.

형식:
- 목적:
- 사용자:
- 필수 요구사항:
- 선택 요구사항:
- 예외/오류:
- 입력:
- 출력:
- 권한/제약:
- 수용 기준:

작성 기준:
- `해야 한다`, `하지 않는다`, `허용한다`처럼 판정 가능한 문장으로 쓴다.
- 예외 조건과 정상 조건을 분리한다.
- 모호한 단어는 확인 질문으로 남긴다.

검증:
- 각 요구사항이 테스트 가능한 문장인지 확인한다.
