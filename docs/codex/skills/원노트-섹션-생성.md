# 원노트 섹션 생성

## 주문번호

SK-003

## Trigger

사용자가 특정 섹션 그룹 아래에 새 섹션을 만들어달라고 할 때 사용한다.

## Instructions

1. `docs/codex/onenote-targets.json`에서 대상의 `section_group_id`를 확인한다.
2. 새 섹션 이름을 파일명으로 쓸 수 있게 정리하고 `.one` 확장자를 붙인다.
3. `OpenHierarchy("섹션명.one", sectionGroupId, ref newId, cftSection)`를 호출한다.
4. `GetHierarchy(sectionGroupId, hsSections, ref xml)`로 새 섹션 이름이 생겼는지 확인한다.
5. 필요하면 `NavigateTo(newId)`로 새 섹션을 화면에 띄운다.

## PowerShell Pattern

```powershell
[void][System.Reflection.Assembly]::LoadWithPartialName("Microsoft.Office.Interop.OneNote")
$one = New-Object -ComObject OneNote.Application

$sectionGroupId = "{2716C2CA-1EA5-4697-9AE7-97380372C026}{1}{B0}"
$sectionName = "새 섹션"
$safeName = $sectionName -replace '[<>:"/\\|?*]', '-'
if (-not $safeName.EndsWith(".one")) {
    $safeName = "$safeName.one"
}

$newSectionId = ""
$one.OpenHierarchy(
    $safeName,
    $sectionGroupId,
    [ref]$newSectionId,
    [Microsoft.Office.Interop.OneNote.CreateFileType]::cftSection
)

$one.NavigateTo($newSectionId)
```

## OneNote COM Guardrails

- 섹션 생성에는 `SectionGroup ID`가 필요하다.
- 같은 이름이 이미 있으면 OneNote 동작이 환경별로 달라질 수 있으므로 생성 전 이름 충돌을 확인한다.
- 검증은 `GetHierarchy(sectionGroupId, hsSections, ref xml)`로 한다.
