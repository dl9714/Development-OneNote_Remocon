# 기능 기획서 정리

## 주문번호

SK-019

## Trigger

사용자가 새 기능, 화면, 자동화, 개선 아이디어를 기획서 형태로 정리해 달라고 요청할 때

## Instructions

목표:
- 기능의 목적, 사용자 흐름, 구현 범위를 명확히 정리한다.

형식:
- 기능명:
- 문제:
- 목표:
- 사용자 시나리오:
- 핵심 동작:
- 제외 범위:
- 화면/입력/출력:
- 검증 기준:
- 열린 질문:

작성 기준:
- 구현 방법보다 사용자에게 보이는 동작을 먼저 쓴다.
- 제외 범위를 반드시 적어 작업 범위를 좁힌다.
- 검증 기준은 테스트 가능한 문장으로 쓴다.

검증:
- 목표, 핵심 동작, 제외 범위, 검증 기준이 모두 있는지 확인한다.
