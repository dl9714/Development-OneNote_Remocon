# 버그 리포트 정리

## 주문번호

SK-018

## Trigger

사용자가 오류, 버그, 이상 동작, 재현 절차를 정리해 달라고 요청할 때

## Instructions

목표:
- 개발자가 바로 확인할 수 있는 버그 리포트 형식으로 정리한다.

형식:
- 제목:
- 환경:
- 기대 동작:
- 실제 동작:
- 재현 절차:
- 빈도:
- 영향도:
- 첨부/로그:
- 임시 대응:

작성 기준:
- 재현 절차는 번호 목록으로 쓴다.
- 모르는 환경 정보는 `확인 필요`로 남긴다.
- 원인 추정은 실제 증상과 분리한다.

검증:
- 기대 동작과 실제 동작이 서로 비교 가능하게 작성됐는지 확인한다.
