# 프로젝트 회고 정리

## 주문번호

SK-016

## Trigger

사용자가 프로젝트, 작업 단위, 릴리즈, 개선 활동을 돌아보고 회고 문서로 정리해 달라고 요청할 때

## Instructions

목표:
- 작업 결과와 배운 점을 다음 실행에 바로 쓸 수 있는 회고로 정리한다.

형식:
- 제목:
- 기간/범위:
- 잘된 점:
- 아쉬운 점:
- 원인:
- 다음 액션:
- 보류할 항목:

작성 기준:
- 감상보다 재현 가능한 사실과 원인을 우선한다.
- 다음 액션은 담당자, 조건, 완료 기준이 보이게 쓴다.
- 불확실한 원인은 가설로 표시한다.

검증:
- 잘된 점, 아쉬운 점, 다음 액션이 모두 분리됐는지 확인한다.
