# 품질 점검 체크리스트

## 주문번호

SK-054

## Trigger

사용자가 문서, 기능, 릴리즈, 작업 결과물을 배포하거나 공유하기 전에 점검 체크리스트를 만들어 달라고 요청할 때 사용한다.

## Instructions

목표:
- 누락, 오류, 위험 요소를 사전에 확인할 수 있는 체크리스트를 만든다.

형식:
- 점검 대상:
- 완료 기준:
- 기능 점검:
- 내용 점검:
- 데이터 점검:
- 권한/보안 점검:
- 호환성 점검:
- 사용자 영향:
- 롤백/복구:
- 최종 승인:

작성 기준:
- 각 항목은 체크 가능한 문장으로 쓴다.
- 위험도가 높은 항목을 먼저 배치한다.
- 확인 불가 항목은 "담당자 확인 필요"로 남긴다.

검증:
- 체크 항목이 실제로 예/아니오로 확인 가능한지 점검한다.
