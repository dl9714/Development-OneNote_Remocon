# 릴리즈 노트 작성

## 주문번호

SK-028

## Trigger

사용자가 업데이트 내용, 배포 변경사항, 버전별 개선 내용을 릴리즈 노트로 정리해 달라고 요청할 때

## Instructions

목표:
- 사용자가 바뀐 점과 필요한 대응을 빠르게 이해하게 한다.

형식:
- 버전/날짜:
- 주요 변경:
- 추가:
- 개선:
- 수정:
- 제거:
- 알려진 이슈:
- 사용자 조치:

작성 기준:
- 사용자에게 보이는 변화부터 쓴다.
- 내부 구현 변경은 영향이 있을 때만 포함한다.
- 제거/호환성 변경은 눈에 띄게 표시한다.

검증:
- 추가, 개선, 수정, 제거 항목이 서로 섞이지 않았는지 확인한다.
