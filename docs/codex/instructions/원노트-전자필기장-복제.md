# 원노트 전자필기장 복제

## 내부 용도

사용자가 OneNote 전자필기장을 같은 구조와 내용으로 복제해 달라고 할 때 사용한다.

## 이름 기준

- 기본 대상 이름은 `코덱스-{원본 전자필기장명}`이다.
- 원본이 `사업체-주식 바이럴`이면 대상은 `코덱스-사업체-주식 바이럴`이다.
- 사용자가 명시적으로 다른 이름 규칙을 지시한 경우에만 그 지시를 우선한다.

## 절차

1. `GetHierarchy('', hsPages, ref xml)`로 원본과 대상 전자필기장을 확인한다.
2. 대상 전자필기장이 없으면 OneNote가 열 수 있는 로컬/동기화 폴더 경로를 확정하고 `OpenHierarchy(targetPath, "", ref newId, cftNotebook)`로 연다.
3. 대상이 이미 있고 사용자가 "똑같이" 또는 덮어쓰기를 요청했다면 대상의 활성 섹션/섹션 그룹을 비운 뒤 다시 복제한다.
4. 원본의 직접 하위 섹션 그룹과 섹션을 재귀적으로 순회한다.
5. `OneNote_RecycleBin` 섹션 그룹과 그 하위 항목은 일반 복제 대상에서 제외한다.
6. 섹션 그룹은 `OpenHierarchy("그룹명", parentId, ref newId, cftFolder)`로 만든다.
7. 섹션은 파일명으로 쓸 수 없는 문자를 정리하고 `OpenHierarchy("섹션명.one", parentId, ref newId, cftSection)`로 만든다.
8. 페이지는 대상 섹션에 빈 페이지를 만든 뒤 원본 페이지 XML의 실제 내용 노드를 대상 페이지 XML로 가져와 `UpdatePageContent`로 반영한다.
9. 복제 후 `GetHierarchy('', hsPages, ref xml)`로 원본과 대상의 활성 섹션 그룹 수, 섹션 수, 페이지 수를 비교한다.

## 페이지 XML 복제 기준

- 대상 페이지는 먼저 `CreateNewPage(destSectionId, ref destPageId, npsBlankPageWithTitle)`로 만든다.
- 원본 페이지와 대상 페이지를 모두 `GetPageContent`로 읽는다.
- 대상 페이지의 `ID`와 `lastModifiedTime`은 유지한다.
- 원본 페이지 루트 속성은 `ID`, `lastModifiedTime`, `selected`, `xmlns:one`을 제외하고 복사한다.
- 원본 페이지의 자식 노드는 XML import 방식으로 대상 페이지에 추가한다.
- `selected`와 `objectID` 속성은 재귀적으로 제거해 OneNote가 새 객체 ID를 만들게 한다.
- `UpdatePageContent` 실패 시 페이지 제목, 원본 Page ID, 대상 Page ID를 함께 기록한다.

## 주의 기준

- 원본 전자필기장이 OneDrive 웹 URL만 보일 수 있으므로 대상 생성은 웹 URL이 아니라 로컬/동기화 경로를 사용한다.
- `OpenHierarchy` 직후 바로 페이지를 만들면 섹션 ID가 아직 준비되지 않을 수 있으므로 재시도와 짧은 대기를 둔다.
- 이름이 같은 전자필기장이 있을 수 있으므로 이름만 보지 말고 경로와 상위 구조도 확인한다.
- 대상 전자필기장 삭제나 비우기는 사용자의 복제 의도가 명확할 때만 한다.

## 검증

- 내부 휴지통을 제외한 활성 섹션 그룹 수가 원본과 대상에서 같아야 한다.
- 내부 휴지통을 제외한 활성 섹션 수가 원본과 대상에서 같아야 한다.
- 내부 휴지통을 제외한 활성 페이지 수가 원본과 대상에서 같아야 한다.
- 필요하면 대표 페이지를 `GetPageContent`로 읽어 제목과 본문 일부가 들어갔는지 확인한다.
