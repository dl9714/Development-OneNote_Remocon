# 원노트 페이지 추가

## 내부 용도

사용자가 OneNote 특정 섹션에 새 페이지를 만들고 제목/본문을 작성해달라고 할 때 사용한다.

## 절차

1. `docs/codex/onenote-targets.json`에서 대상 경로와 `section_id`를 먼저 확인한다.
2. `section_id`가 있으면 `GetHierarchy` 전체 탐색 없이 바로 `CreateNewPage`를 호출한다.
3. `GetPageContent`로 새 페이지 XML을 읽고 제목과 본문을 수정한다.
4. `UpdatePageContent`로 반영하고 `GetPageContent`로 검증한다.
5. 사용자가 화면 확인을 원하면 마지막에 `NavigateTo(pageId)`를 호출한다.

## PowerShell Pattern

```powershell
[void][System.Reflection.Assembly]::LoadWithPartialName("Microsoft.Office.Interop.OneNote")
$one = New-Object -ComObject OneNote.Application

$sectionId = "{175CFD85-2C5C-0C63-3116-2598A37ACB11}{1}{B0}"
$title = "새 페이지"
$body = "본문"
$pageId = ""

$one.CreateNewPage(
    $sectionId,
    [ref]$pageId,
    [Microsoft.Office.Interop.OneNote.NewPageStyle]::npsBlankPageWithTitle
)

$pageXml = ""
$one.GetPageContent($pageId, [ref]$pageXml)

# XML의 Title/Outline을 수정한 뒤 반영한다.
$one.UpdatePageContent($pageXml)
$one.NavigateTo($pageId)
```

## 주의 기준

- 화면 클릭 자동화보다 OneNote COM API를 우선 사용한다.
- `hsPages` 전체 조회는 느리므로 기본 작업에서는 피한다.
- 검증은 화면 캡처보다 `GetPageContent` 결과로 한다.
