# 원노트 대상 ID 찾기

## 내부 용도

새 OneNote 작업 위치를 자주 쓰게 되었거나 캐시된 ID가 비었거나 실패할 때 사용한다.

## 입력

- 전자필기장 이름
- 섹션 그룹 이름 또는 상위 경로
- 섹션 이름
- 필요하면 사람이 읽는 전체 경로

## 절차

1. 기존 `docs/codex/onenote-targets.json`과 `docs/codex/onenote-location-cache.json`을 먼저 확인한다.
2. 캐시에 ID가 없거나 실패하면 `GetHierarchy('', hsSections, ref xml)`로 계층을 조회한다.
3. 전자필기장, 섹션 그룹, 섹션 이름을 상위 경로와 함께 비교해 대상 노드를 고른다.
4. 찾은 ID를 작업에 바로 쓰고, 반복 대상이면 캐시에 저장한다.

## 주의

- 이름만으로 고르지 않는다. 같은 이름의 섹션이나 그룹이 있을 수 있다.
- ID 찾기에는 기본적으로 `hsPages`를 쓰지 않는다.
- 캐시는 힌트다. ID 호출이 실패하면 계층 조회로 다시 검증한다.

## 검증

- 선택한 노드의 `ID`, `name`, 상위 경로가 사용자 요청 대상과 일치해야 한다.
- 섹션 작업이면 `section_id`, 섹션 생성 작업이면 `section_group_id`가 비어 있지 않아야 한다.
